# YoYoung Shorts 优扬短剧功能说明

这份页面不是首页概览，而是把主要功能区的价值、使用方式和产品意义讲得更清楚。

YoYoung Shorts 优扬短剧更像一套围绕真实创作流程组织出来的短剧工作台，而不是“模型入口列表”。这一点很重要，因为现在 GitHub 上并不缺能调用模型的工具，真正稀缺的是：

- 让不会写复杂提示词的人也能开始
- 让角色、场景、道具在后续阶段继续起作用
- 让脚本、图片和视频之间形成连续关系
- 让结果不是一次性产物，而是能继续沉淀、回看和复用

如果用一句话概括，它更像：

> 从一句想法出发，逐步组织成分镜脚本、连续图片和视频结果的创作工作台。

## 为什么不是“又一个模型壳”

很多同类产品更强调：

- 接了多少模型
- 支持多少接口
- 能不能快速出一张图

而 YoYoung Shorts 优扬短剧更强调：

- 创作流程是不是连着的
- 图片一致性是不是能持续被感知
- 角色、场景、道具是不是能被当成资产继续使用
- 脚本和图片能不能继续推进到视频结果

所以它的重点不是“堆能力清单”，而是把创作链真正串起来。

## Create：从一句想法开始

Create 不是传统意义上的“提示词输入框”，而更像是整个创作流程的起稿区。

它解决的问题是：很多人其实并不会写复杂提示词，但他们有一个故事想法、一个角色关系、一个氛围方向，甚至只是一句模糊描述。YoYoung Shorts 优扬短剧的目标，不是要求用户先成为提示词工程师，而是让用户先把创意说出来，再逐步推进成脚本、画面和视频。

这个区块最重要的价值在于：

- 降低开始创作的门槛
- 把灵感尽快转换成可继续组织的内容起点
- 为后续 Story、Script、Video 提供统一入口

## Story：多图叙事不是重复同一张图

Story 区不是“多生成几张图”这么简单。

它真正要表达的是：同一个剧情段落可以被组织成多张连续画面。这些画面之间不只是风格接近，而是存在角色关系、镜头变化、动作推进和叙事节奏上的连续感。

这一块的价值主要体现在：

- 把一个段落扩展成连续画面，而不是孤立图片
- 保持角色、场景、道具在多张图里的延续性
- 为正式分镜脚本提供更明确的视觉参考

如果说单图更像“一个结果”，那么 Story 更像“一个正在展开的片段”。

## Script：自动组织系列化分镜脚本

Script 区是产品里最能体现“工作流价值”的部分之一。

它不是只生成一段文案，而是会把内容组织成可继续执行的不同分镜脚本。每一个镜头都可以有自己的画面描述、动作提示、对白与旁白，并继续连接到图片和视频生成阶段。

这里真正重要的地方在于：

- 把故事从“想法”推进到“可执行镜头”
- 让脚本不是终点，而是后续画面和视频的中间骨架
- 降低从文本构想到可视化表达的断层

对于不会写专业分镜的人，这一块尤其重要，因为它把原本需要经验的镜头组织，变成了更可进入的流程。

## Video：从图片继续推进到视频

Video 区的意义，不只是“支持视频生成”。

更重要的是，它说明这套工具不是停留在静态图层，而是可以把已经形成的画面和镜头继续推进到视频阶段。也就是说，图片不是终点，脚本也不是终点，它们都可以继续向成片结果靠近。

这一块当前最值得强调的价值是：

- 承接前面已经生成的图片和脚本结果
- 把静态创作继续推进到视频结果
- 让“图 -> 视频”形成真实可见的闭环

这会让整套产品更像创作工作台，而不是一组分散的生成按钮。

## Assets：角色、场景、道具的锚定系统

Assets 区是理解 YoYoung Shorts 优扬短剧和普通生成工具差异的关键。

很多工具的问题是：每次生成都像重新开始，角色、场景、道具只能靠重新写提示词维持。资产中心要解决的，是让这些内容真正成为工作流中的“锚点”。

它的价值包括：

- 角色可以持续进入后续单图、多图、分镜和视频流程
- 场景不是背景词，而是可复用的创作资产
- 道具能够参与连续创作，而不只是临时描述

这意味着创作不再是“一次性输入 -> 一次性输出”，而是“资产持续参与 -> 内容持续延展”。

## History：结果不是一次性的

History 区不是简单的回收站。

它的存在很重要，因为它证明这不是一个“生成一次、结果就结束”的工具。无论是图片还是视频，结果都能沉淀下来，被继续回看、比较、复用和扩展。

它在产品层面的价值是：

- 让创作过程可追溯
- 让用户能回看不同阶段的结果
- 让内容可以继续二次利用，而不是每次从零开始

在公开展示层面，History 也是非常重要的说明页，因为它能直接体现“结果是连续积累的”。

## 这套工作流真正想表达什么

YoYoung Shorts 优扬短剧最重要的不是单个按钮多强，而是这些功能区彼此之间是连着的：

`Create -> Story -> Script -> Video -> History`

同时，`Assets` 会持续进入这条链路中。

所以它不是一个单点 AI 工具，而是一套更接近真实创作流程的工作台。
